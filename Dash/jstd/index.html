JsTestDriver

  1. Test execution server created by google

  2. Runs on Java

  3. Allows to execute test automatically across multiple clients

  4. Has a GUI interface for most IDEs (eclipse, JetBrains etc)

  5. Has a basic, prototype based test framework

Jasmine

  1. Based on RSpec

  2. Consists of readable stories rather than programable syntax

  3. Has a robust and flexible set of tools

  4. Can be integrated with almost any invironment (JSTD, Node, etc)

Conclusion

  1. Testing is fun. JSTD+Jasmine makes it easy

  2. We want to test JS logic, not CSS behavior

  3. Having an Event-based code makes it much easier to write async-tests

  4. No excuse not to start NOW
